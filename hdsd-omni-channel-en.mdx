# **Chatbot module user guide - Jinmi Omnichannel**

## **Part 1: Register/log in to your account**

### **1. Register an account**

Step 1: On the registration screen → select **_Continue with Google_**

Step 2: Log in with your Google account

Step 3: Set up your store information → Click **_Continue_**

You have now completed registration for Jinmi Omnichannel.

### **2. Log in**

There are two ways to log in to the Omni platform:

- <u>Method 1: Log in with your account</u>

Step 1: Go to the link: [<u>Jinmi Portal</u>](https://portal.jinmi.ai/auth?return_url=https%3A%2F%2Fportal.jinmi.ai%2Fapi%2Foauth%2Fauthorize%3Fclient_id%3Domnichannel_client_id%26redirect_uri%3Dhttps%253A%252F%252Fomni.jinmi.ai%252Fportal%252Fsso%252Fcallback%26response_type%3Dcode%26scope%3Dopenid%2Bprofile%2Bemail%26state%3DLw%253D%253D)

Step 2: Enter the registered email and a valid password, then click "Log in"

\

- <u>Method 2: Log in with Google</u>

Step 1: On the login screen, click "Log in with Google"

Step 2: Select the account you want to log in with

You have now successfully logged in to the Jinmi Omnichannel system.

## **Part 2: Connect LINE to Omnichannel**

### **1. Create a LINE Official Account**

Step 1: Select **_Create a LINE Official Account_**

Step 2: Enter the required information → Select **_Continue_**

Step 3: Review the information → Select **_Submit_**

Step 4: When **_Application Complete_** is displayed → select **_Go to LINE Official Account Manager_**

### **2. Enable Messaging API**

Step 1: The main screen will display Welcome => select **_Next_** => Then select **_Home Screen_** to go to the home page.

Step 2: Select **_Settings_** → Select **_Messaging API_**

Step 3: Click **_Enable Messaging API_**

Step 3: Enter the **_Account Name_** → Select **_Agree_**

Step 4: Enter the required information → Select **_OK_**

### **3. Get connection details**

Step 1: Go to **_LINE Developers Console_**

Step 2: On the **_Provider_** screen, select the connection channel

Step 3: On the channel connection screen, select **_Messaging API_**

Step 3: Then scroll down and select **_Issue_**

### **4. Connect LINE to Jinmi**

Step 1: Go to **_Jinmi Omnichannel_** → Select **_Settings_** → **_Inboxes_**

Step 2: Select **Add Inbox**

**Step 3:** Select **_LINE_** and enter the channel information

Go back to the **_LINE_** screen → Copy each item: **_Channel ID, Channel Secret, Token_** to fill in the channel information (LINE Channel ID, LINE Channel Secret, LINE Channel Token)

Step 3: Select **_Create LINE Channel_**

Step 4: Add **_Agents_** (staff members) → Select **_Add Agents_**

Step 5: Copy the **_Webhook URL_** from the Jinmi system

Step 6: Go back to LINE → Paste the **_Webhook URL_** → Select **_Save_**

Step 7: In **_Settings_** → Select **_Response settings_** → Enable **_Chat, Webhook_**

Step 8: LINE is now successfully connected. Go back to Omnichannel → select **_More Settings_**

You have now completed connecting LINE to Omnichannel and it is ready to use. Agents can now chat with customers via LINE.

**Part 3: How to create and train an AI chatbot**

### **1: Create a chatbot**

Step 1: Go to **_Jinmi Agent_** → Select **_Create a New Assistant_**

Step 2: Enter the following information:

1. Enter the chatbot name (_e.g.: Chatbot for product consultation, answering frequently asked questions, and 24/7 customer support_)
2. Describe the chatbot's role
3. Main product (_e.g.: Jinmi chatbot AI_)
4. Select features based on your needs to choose the appropriate bot features
5. Set the resolution message sending time
6. Click **_Create_**

### **2. Train the chatbot**

- Create FAQs: a list of frequently asked questions and pre-set answers.

The chatbot will use these question-answer pairs to automatically respond to customers, saving time for agents.

Step 1: After creating the chatbot => click **_FAQS_** => click **_Create new FAQ_**

Step 2: Add FAQ:

1. Enter a question that customers frequently ask (_e.g.: Does this product have a warranty?_)
2. Enter the answer
3. Select the chatbot that should use this question
4. Add images (if any)
5. Click **_Create_**

- Attach documents to the chatbot

Jinmi Documents is where you store information, articles, and guides for the chatbot to read and learn from.

The chatbot will use these documents to answer customers more accurately.

Step 1: In **_Jinmi Agent_** => Select **_Documents_**

\

Step 2: Select **_Create a new document_**

Step 3: Enter the information

1. Enter the document name (_e.g.: Omni user guide_)
2. Select the document type
3. Paste the document link
4. Select the chatbot
5. Click **_Save_**

### **3. Connect the chatbot to a channel**

Step 1: Select **_Jinmi Agent_** → On the chatbot you want to connect to LINE, click the (...) icon → Select **_Connect Inbox_**

Step 2: Select **_Connect a new inbox_**

Step 3: Select the channel you want to connect (Facebook/Zalo/Website/WhatsApp/Instagram…)

Step 4: Click **_Create_** to complete the inbox connection

At this point, the chatbot can:

- Automatically respond to customers
- Understand content based on documents
- Support multiple languages

**Note:** You only need to create documents for the chatbot in one language. When customers send messages in another language, the chatbot will still understand and reply in the customer's language.

**4. Test the chatbot**

- Method 1: Test the chatbot directly on Jinmi Omnichannel

Step 1: Go to **_Jinmi Agents_** → On the chatbot you want to test, click the (...) icon → Select **_Edit Assistant_**

\
**	**Step 2: Select **_Playground_** → Enter a message to test the chatbot's response

- Method 2: Chat as a customer: Use a personal account → send a message to your page/LINE

You will see the chatbot automatically respond based on the trained content. If the bot does not answer correctly, you can add more FAQs or documents.

**Part 4: Conclusion**

You have now completed creating, training, and deploying a chatbot on Jinmi Omnichannel.

The chatbot will automatically assist with consulting and responding to customers, helping you save time and optimize customer support efficiency.
