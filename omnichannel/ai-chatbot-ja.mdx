Jinmi AIチャットボットは、顧客の質問に対応し、商品を推薦し、自動的に受注処理を行います — いつでも、どのチャネルでも。スタッフの返信を待つ代わりに、チャットボットにトレーニングした情報に基づいて、顧客は即座に正確な回答を受け取ります。

<Note>
  Luna Boutiqueは、AIチャットボット導入後3か月で売上27%増を記録。JUNI Careは応答時間を2時間から10秒に短縮し、人件費を40%削減しました。
</Note>

## 主な機能

### 常時稼働の自動応答

チャットボットは夜間、週末、祝日を含む24時間365日稼働します。営業時間外にメッセージを送った顧客は、翌朝まで待つことなく即座に返信を受け取ります。これはEコマースにおいて特に重要で、顧客の60%以上が10分以内に返信がなければ購入を断念します。

### 自然言語理解

チャットボットは、くだけた表現、タイプミス、さまざまな質問の仕方を含む、顧客の自然な書き方を理解します。特定のキーワードの使用を求めません。「これの青はありますか？」や「返品ポリシーは？」といった質問でも、意図を理解して適切に回答します。

### トレーニングファイル

商品、サービス、ポリシー、販売スクリプトを記述したファイルをアップロードすることで、チャットボットに「教える」ことができます。チャットボットはこれらのファイルを読み取り、顧客の質問に正確に回答するために使用します。新商品や変更されたポリシーを反映するため、トレーニングファイルはいつでも更新可能です。

対応するトレーニングファイルの種類：商品カタログ、FAQドキュメント、価格表、販売会話スクリプト。

### スクリプトのカスタマイズ

自由形式の応答に加えて、スクリプトを使用して構造化された会話フローを定義できます。スクリプトは、顧客の返答に基づいて、リードの選別、注文の受付、予約の設定など、特定のパスにチャットボットを誘導します。

### 多言語対応

チャットボットは複数の言語で顧客に対応できます。異なる市場で事業を展開している場合や、チームとは異なる言語を使用する顧客にサービスを提供する場合に便利です。

### 統合受信トレイとの連携

チャットボットでは対応しきれない複雑な質問や、顧客が人間のオペレーターを要求した場合、会話はオムニチャネル受信トレイのチームに直接転送されます。完全な会話履歴が保持されるため、オペレーターは必要なすべてのコンテキストを把握できます。

## チャットボットが対応するタスク

<Tabs>
  <Tab title="FAQ">
    よくある質問に自動回答 — 価格、配送時間、返品ポリシー、在庫状況、営業時間。FAQドキュメントをトレーニングファイルとしてアップロードするだけで、チャットボットがすぐに学習します。
  </Tab>
  <Tab title="注文に関する問い合わせ">
    注文状況、追跡番号、配送予定日について質問する顧客に即座に回答。注文データを連携することで、リクエストに応じてチャットボットが特定の注文情報を検索可能です。
  </Tab>
  <Tab title="商品推薦">
    チャットボットがサイズ、予算、用途などの質問をし、カタログから適切な商品を推薦。経験豊富な販売スタッフとの会話のような体験を再現します。
  </Tab>
  <Tab title="リード選別">
    スクリプトを使用して連絡先情報を収集し、営業チームに転送する前に見込み客を選別。チャットボットは名前、連絡先、予算、購入予定時期を自動的に収集できます。
  </Tab>
</Tabs>

## チャットボットの設定方法

<Steps>
  <Step title="チャットボットを作成">
    ステップ1：オムニチャネルダッシュボードで**_チャットボット管理_** → **_新しいチャットボットを作成_**を選択

    ステップ2：以下の情報を入力：

    1. チャットボット名を入力（_例：商品相談、FAQ回答、24時間カスタマーサポート用チャットボット_）
    2. チャットボットの役割を説明
    3. メイン商品（_例：Jinmi chatbot AI_）
    4. ニーズに応じてボットに適した機能を選択
    5. 解決メッセージのタイミングを設定
    6. **_作成_**をクリック
  </Step>
  <Step title="FAQを作成">
    チャットボットはこれらの質問と回答のペアを使用して顧客に自動応答し、スタッフの時間を節約します。

    ステップ1：チャットボット作成後 → **_FAQ_** → **_新しいFAQを作成_**を選択

    ステップ2：FAQを追加：

    1. 顧客がよく聞く質問を入力（_例：この商品に保証はありますか？_）
    2. 回答を入力
    3. この質問を使用するチャットボットを選択
    4. 画像を追加（該当する場合）
    5. **_作成_**をクリック

    <Tip>
      最もよくある質問から始めましょう。過去の会話を振り返り、チームが繰り返し回答している上位20の質問を特定してください — これが最初のトレーニング優先事項です。
    </Tip>
  </Step>
  <Step title="チャットボットにドキュメントを添付" stepNumber={3}>
    Jinmiドキュメントは、チャットボットが読んで学習するための情報、記事、ガイドを保存する場所です。

    チャットボットはこれらのドキュメントに基づいて、より正確に顧客に回答します。

    ステップ1：**_チャットボット管理_** → **_ドキュメント_**を選択

    ステップ2：**_新しいドキュメントを作成_**を選択

    ステップ3：情報を入力

    1. ドキュメント名を入力（_例：Omniユーザーガイド_）
    2. ドキュメントタイプを選択
    3. ドキュメントリンクを貼り付け
    4. チャットボットを選択
    5. **_保存_**をクリック
  </Step>
  <Step title="応答とスクリプトを設定">
    **スクリプト**エディタを使用して、特定のシナリオ向けの構造化された会話フローを定義します — 例えば、予約フローや商品検索フロー。チャットボットが回答できない場合のフォールバックメッセージを設定し、人間のオペレーターへの引き継ぎタイミングを設定します。
  </Step>
  <Step title="引き継ぎルールを設定">
    チャットボットが会話をチームに転送する条件を定義します — 例えば、顧客が特定のキーワードを使用した場合、人間のオペレーターを要求した場合、またはチャットボットの信頼度がしきい値を下回った場合。
  </Step>
  <Step title="チャットボットをチャネルに接続">
    ステップ1：オムニチャネルダッシュボードで**_チャットボット管理_** → **_受信トレイ_**セクションをクリック

    ステップ2：**_新しい受信トレイを接続_**を選択

    ステップ3：接続するチャネルを選択（Facebook/Zalo/ウェブサイト/WhatsApp/Instagram...）

    ステップ4：**_作成_**をクリックして受信トレイの接続を完了
  </Step>
  <Step title="チャットボットをテスト">
    内蔵のテストコンソールを使用して顧客との会話をシミュレーションします。よくある質問に正しく回答するか、オペレーターへの引き継ぎが期待通りに動作するか確認してください。

    チャットボットのテストには2つの方法があります：

    **_方法1_**_：**Jinmiオムニチャネルで直接テスト**_

    ステップ1：**_チャットボット管理_** → テストしたいチャットボットの**(...)アイコン** → **_チャットボットを編集_**を選択\
    ステップ2：**_試す_** → メッセージを入力してチャットボットの応答を確認

    **_方法2：顧客としてチャット_**

    個人アカウントを使用 → チャネルにメッセージを送信。トレーニングされたコンテンツに基づいてチャットボットが自動応答するのを確認できます。チャットボットが正しく応答しない場合は、FAQやドキュメントを追加できます。
  </Step>
  <Step title="運用開始">
    選択した受信トレイでチャットボットを有効化。新しい顧客メッセージに即座に自動応答を開始します。
  </Step>
</Steps>

## トレーニングファイルのベストプラクティス

| 推奨 | 避けるべきこと |
| --- | --- |
| 明確でわかりやすい言葉で書く | 顧客が理解できない社内用語を使用 |
| よくある質問のさまざまな表現を含める | 古い価格や商品情報のファイルをアップロード |
| 商品説明を具体的に（サイズ、素材、バリエーション） | 不完全な回答につながる情報の空白を残す |
| 商品やポリシーの変更時にファイルを更新 | 画像のみのPDFを使用（チャットボットはテキストを読む必要があります） |

## よくある質問

<AccordionGroup>
  <Accordion title="チャットボットはすべての顧客の質問に回答できますか？">
    チャットボットは、提供したトレーニングファイルとスクリプトに基づいて回答します。トレーニングデータの範囲外の質問はフォールバックメッセージをトリガーし、設定されている場合は人間のオペレーターに引き継ぎます。トレーニングファイルが充実しているほど、チャットボットのカバー範囲は広がります。
  </Accordion>

  <Accordion title="スクリプトの長さや数に制限はありますか？">
    スクリプトの制限は料金プランによって異なります。Freeプランにはトレーニングファイル3件、Basicは5件、Advancedは15件、Professionalは30件、Enterpriseは無制限です。
  </Accordion>

  <Accordion title="チャットボットは注文処理と決済ができますか？">
    チャットボットは注文情報を収集し、購入フローを通じて顧客をガイドできます。決済処理との連携は設定によって異なります。詳細はJinmiチームにお問い合わせください。
  </Accordion>

  <Accordion title="チャットボットはセールススクリプトに対応していますか？">
    はい。カスタムのセールススクリプトをアップロードし、構造化された会話フローを定義できます。チャットボットはこれらのスクリプトに従って、リードの選別、アップセル、予約ガイドを行います。
  </Accordion>
</AccordionGroup>
